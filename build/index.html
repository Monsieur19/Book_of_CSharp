1.1 История и особенности языка

Тема 1/4 Основы синтаксиса и структуры языка ➜ Урок 1/3

Краткая история языка C# (создатели, год появления, развитие)

C# («Си Шарп») — объектно-ориентированный язык программирования, созданный компанией Microsoft в рамках платформы .NET.

Разработка велась с 1998 по 2002 год под руководством Андерса Хейлсберга — известного разработчика Turbo Pascal и Delphi. Вместе с ним работали Скотт Вильтаумот и другие инженеры Microsoft. Целью было создание современного, безопасного и удобного языка, сочетающего лучшие черты C++, Java и других языков, но при этом лишённого их недостатков.

НА ЗАМЕТКУ!
C# задумывался как альтернатива Java, на который у Microsoft не было лицензии. При этом C# стал более тесно интегрирован с Windows и экосистемой Microsoft.

C# официально был представлен в 2002 году, и с тех пор язык активно развивается. Он стал частью платформ .NET Framework, а затем — кроссплатформенных решений .NET Core и .NET 5+, что позволило запускать программы на Windows, Linux и macOS.

НА ЗАМЕТКУ!
Название «C#» символизирует следующий шаг в развитии C-подобных языков. Решётка (#) визуально напоминает четыре знака «плюс» — как бы продолжение C++.

Отличия C# от других языков (Java, C++, Python)

C# унаследовал синтаксис и концепции от C++ и Java, но при этом имеет собственную философию и функциональность:

  • Как и Java, C# предлагает автоматическое управление памятью через сборщик мусора.
  • В отличие от C++, C# исключает ручное управление памятью и потенциально опасные конструкции, повышая безопасность кода.
  • По сравнению с Python, C# — строго типизированный язык, что предотвращает ошибки уже на этапе компиляции.
  • C# предлагает современные средства разработки, такие как LINQ, async/await, делегаты и обобщённые типы, чего нет в Java и C++ в базовой поставке.
НА ЗАМЕТКУ!
Синтаксис C# будет знаком программистам на C++ и Java, но это не делает языки одинаковыми. Важно изучать особенности C# отдельно — он имеет собственные принципы и преимущества.

Сегодня C# — универсальный язык, применяемый во многих областях:

1. Разработка программного обеспечения (ПО)

  • Десктопные приложения для Windows (с помощью Windows Forms, WPF, MAUI).
  • Инструменты автоматизации и корпоративные решения.

2. Веб-разработка

  • C# используется в связке с ASP.NET для создания надёжных и производительных веб-сервисов, сайтов и API.

3. Разработка игр

  • Является основным языком в Unity — одном из самых популярных игровых движков. Подходит для 2D/3D-игр на ПК, консолях и мобильных устройствах.

4. Мобильные приложения

  • Благодаря Xamarin и .NET MAUI можно писать кроссплатформенные приложения под Android и iOS.

НА ЗАМЕТКУ!
Одна из сильных сторон C# — кроссплатформенность. С появлением .NET Core и .NET 6/7 разработка на C# возможна не только под Windows, но и под Linux/macOS, что делает его конкурентом языкам, традиционно считавшимся более универсальными.

Подробный разбор каждой строки

1. using System;

Структура программы: using, namespace, class, Main()

Чтобы создавать более сложные приложения, важно понимать, как устроена простейшая программа.

Основные элементы:

1. using

  • Подключает внешние пространства имён, в которых определены нужные классы.
  • Например: using System; подключает Console, Math, DateTime и другие полезные классы.

2. namespace

  • Пространство имён (англ. "namespace") — это способ организации кода.
  • Помогает избежать конфликтов имён, когда в проекте много классов.

2. namespace

  • Пространство имён (англ. "namespace") — это способ организации кода.
  • Помогает избежать конфликтов имён, когда в проекте много классов.

3. class

  • Класс — это основная единица, внутри которой описываются все действия.
  • Название класса должно начинаться с заглавной буквы.
  • Обычно именуется как Program в базовых примерах.

4. Main()

  • Это точка входа — программа начинается с выполнения команд внутри Main.
  • static — означает, что метод принадлежит классу, а не его объекту.
  • void — метод не возвращает значение.
  • string[] args — можно передавать параметры из командной строки (чаще используется в профессиональных приложениях).

Все ключевые слова (class, static, void, using, namespace) — это специальные зарезервированные слова языка C#.

Программа обязательно должна содержать метод Main — без него она не скомпилируется.

2.1 Типы данных и переменные

Примитивные типы данных (int, double, bool, char)

В языке C# тип переменной определяет, какие данные она может хранить. Например, целое число, дробное, символ или логическое значение.

Примитивные типы — это фундаментальные типы данных в C#, с помощью которых описываются наиболее простые значения: числа, символы, логические значения. Они называются примитивными, потому что не состоят из других типов и хранят значение прямо в памяти, а не ссылку на него.

  • int — целые числа (например: 5, -100, 0).
  • double — дробные числа с плавающей точкой (например: 3.14, -0.5).
  • bool — логические значения: true (истина) или false (ложь).
  • char — одиночный символ (например: 'A', '7', '#').

Обратите внимание, что в переменных типа double значение записывается через точку, а не через запятую, как принято в математике. Так, double temperature = 36.6; — корректная запись, а 36,6 вызовет ошибку. Этот тип поддерживает высокую точность, поэтому подойдёт для математических вычислений и финансов.

Логические значения в C# описываются типом bool. Это сокращение от слова "boolean", что значит "логический". Этот тип незаменим при создании условий: например, включен ли пользователь, закончилась ли игра, получен ли доступ. Когда вы пишете bool isRegistered = true; , это значит, что пользователь зарегистрирован.

НА ЗАМЕТКУ!
Все примитивные типы занимают определённое количество памяти: int — 4 байта, double — 8 байт, bool — 1 байт, char — 2 байта.

Какие из следующих строк кода корректно создают переменные примитивных типов в языке C#?

Ссылочные типы данных (string, object)

В отличие от примитивных типов, ссылочные типы не хранят само значение, а хранят ссылку на область памяти, где это значение находится.

Два самых часто используемых ссылочных типа:

  • string — строка текста, заключённая в двойные кавычки.
  • object — универсальный тип, родительский для всех других типов.

Тип string используется для хранения строк — последовательностей символов. Текст любого содержания, как короткие слова, так и целые абзацы, можно хранить в переменной типа string . Строки заключаются в двойные кавычки: "Привет" или "Hello, World!". Внутри строки можно использовать пробелы, спецсимволы и даже управляющие символы, например \n для новой строки.

Тип object является базовым для всех других типов. Это значит, что любой тип в C#, включая примитивные, может быть сохранён в переменной типа object. Однако, чтобы использовать данные в object-переменной, их нужно будет преобразовать обратно в нужный тип. Это делает object универсальным, но не самым удобным для повседневных задач. Он особенно полезен, когда вы не знаете заранее, какой тип данных будет использоваться.

Когда вы пишете string userName = "Анна"; , это значит, что в переменной userName хранится строка. А запись object box = 123; сохраняет в переменной box целое число, но через обобщённую обёртку. Впоследствии потребуется привести box обратно к int , если вы захотите выполнить арифметические операции.

НА ЗАМЕТКУ!
Тип string — это на самом деле класс, поэтому у строк есть методы, которые можно вызывать: ToUpper(), Length, Contains() и другие.

В программировании переменная — это как коробка или ячейка, в которую можно положить данные. Объявление переменной означает, что вы создаёте эту коробку. При этом вы указываете тип данных и придумываете имя переменной. Например, строка int counter; означает, что в программе появится переменная counter , в которой будут храниться целые числа.

Если сразу известно значение, переменную можно и объявить, и проинициализировать одновременно: int counter = 10;. Это удобнее, особенно в простых случаях. Если же значение станет известно позже — достаточно сначала объявить переменную, а потом отдельно присвоить ей значение:

Константы и неизменяемые значения

В некоторых случаях необходимо, чтобы переменная никогда не изменялась после присваивания. Например, математическая константа Пи ( 3.14 ) или максимально допустимое количество попыток входа в систему. В C# для этого используется ключевое слово const.

Константа объявляется так же, как обычная переменная, но с ключевым словом const перед типом. Константы всегда нужно инициализировать при объявлении. Иначе возникнет ошибка компиляции.

Если вы попробуете изменить значение константы, программа не скомпилируется. Это защищает от случайных ошибок. Константы — хороший способ сделать код более понятным и надёжным: const double gravity = 9.8 ; сразу даёт понять, что значение не изменится.

НА ЗАМЕТКУ!
Константам тоже принято давать осмысленные имена. Обычно их записывают в PascalCase, начиная каждое слово с заглавной буквы, например MaxSpeed или PiValue.

2.2 Операторы и выражения

Арифметические, логические и побитовые операторы

В программировании операторы — это символы, которые обозначают действие. Они используются в выражениях, где обрабатываются значения переменных.

Арифметические операторы позволяют выполнять математические действия. В C# доступны сложение +, вычитание -, умножение *, деление / и остаток от деления %. Например, выражение int sum = 5 + 3; вернёт значение 8. Деление двух целых чисел даёт целый результат, поэтому 7 / 2 вернёт 3, а не 3.5. Чтобы получить дробь, хотя бы одно число должно быть типа double.

Логические операторы применяются к значениям true и false. Оператор && возвращает истину, если оба условия истинны. Оператор || возвращает истину, если хотя бы одно условие истинно. Оператор ! меняет логическое значение на противоположное. Например, если bool isRain = false;, то !isRain будет true.

Побитовые операторы работают с двоичными представлениями чисел. Они используются реже, чаще в системном и низкоуровневом программировании. К ним относятся &, |, ^, ~, <<, >>. Эти операции применяются побитово и полезны, например, при работе с флагами или масками доступа.

НА ЗАМЕТКУ!
Логические операторы работают только с bool. Побитовые операторы можно применять к числам типа int, byte, long и другим целочисленным типам.

Что вернёт выражение (5 > 3) && (2 == 2)?

Операторы присваивания и сравнения

Оператор присваивания — это символ =, он используется для того, чтобы сохранить значение в переменную. Например, int age = 25; — создаёт переменную age и присваивает ей значение 25.

Кроме обычного присваивания, есть и составные операторы присваивания, которые совмещают арифметику и присваивание. Например, x += 5; означает "прибавить 5 к значению x" и сохранить результат в x. Это более короткая форма записи x = x + 5;.

Операторы сравнения возвращают значение типа bool — true или false. Они проверяют, равны ли значения (==), не равны (!=), больше (>), меньше (<), больше или равно (>=) и меньше или равно (<=). Например, 7 > 3 вернёт true, а 5 == 10 — false.

НА ЗАМЕТКУ!
Не путайте = и ==. Первый присваивает значение, второй сравнивает. Ошибка в использовании этих операторов — одна из самых частых у новичков.
